# Architecture & Tech Stack

## 1. High-Level Architecture
The application now follows a **Hybrid Local-First Study Architecture**:

- **Static content** is still shipped as pre-ingested JSON and loaded directly in the browser.
- **Study data** (notes, highlights, reading state) can run in local-only mode or sync through Firebase.
- **The UI stays local-first** by keeping content fetches and reading interactions fast even when auth/sync is enabled.

## 2. Platform: Local-First Web App (PWA)

### Why a Web App over Native?
- No $99/yr Apple Developer fee to sign and distribute.
- Write entirely in React/TypeScript — no Swift/C# learning curve.
- "Install" to desktop dock via PWA, use completely offline.
- Bypass Apple's developer gatekeeping and scary unsigned-app warnings.

### Achieving Native Feel
- Advanced CSS (`backdrop-filter: blur()`) for glassmorphism effects.
- Radix UI headless components for native-feeling menus, dropdowns, and dialogs.
- Keyboard-first interaction design (`Cmd+K` palette, shortcuts for all actions).

## 3. Storage Layer

### Static Content: JSON Manifests + Chapter Documents
Scripture and study-edition content lives as pre-ingested JSON files in `data/` (generated by `scripts/ingest.ts`). These are loaded at runtime via `fetch()` and normalized in `src/lib/scripture.ts`.

### Study Data: Local Repository Or Firebase Sync
Study data now goes through a repository abstraction instead of direct component-level `localStorage` access.

- **Local mode:** highlights, notes, reading state, and imported private dictionary/index data live in `localStorage`.
- **Cloud mode:** Google OAuth via Firebase Authentication, plus synced study data in Firestore with offline cache support.
- **Current repository target:** Highlights, notes, reading state, and tags.

See `src/context/AuthContext.tsx`, `src/lib/firebase.ts`, and `src/lib/studyRepository.ts`.

### Future: SQLite WASM + OPFS
When we need heavier local search, very large study datasets, and richer index/query support:
- **SQLite WASM** compiled to WebAssembly, running in a Web Worker.
- **OPFS** (Origin Private File System) for large persistent browser-local storage.
- **Schema direction:** Blocks, Highlights, Notes, Tags, Lessons, Variants, Dictionaries, Indices (see `src/db/db.ts`).

### User Notes: File System Access API
The `useFileSystem.ts` hook provides read/write access to a user-selected local directory. Notes and exports are stored as transparent Markdown files the user owns and can edit externally.

## 4. Data Flow

```
EPUB files → scripts/ingest.ts → data/<profile>/*.json
                                        ↓
                          scripts/generate-manifest.ts → data/manifest.json
                                        ↓
                          public/data (symlink) → Vite serves as static files
                                        ↓
                    src/lib/scripture.ts (fetch + normalize + cache) → React components
                                                                         ↓
                              AuthProvider + StudyRepository abstraction decide persistence
                                                                         ↓
                            localStorage (local mode) or Firestore + offline cache (sync mode)
                                                                         ↓
                                                   SQLite WASM + OPFS (future heavy local query layer)
```

## 5. Auth & Sync

- **Identity:** Google OAuth through Firebase Authentication.
- **Synced entities:** highlights, notes, and reading state.
- **Offline behavior:** Firestore local cache plus optimistic UI updates keep annotation workflows usable offline.
- **Fallback:** if Firebase env vars are absent, the app remains fully usable in local-only mode.

Detailed flow: see `docs/architecture/auth_sync.md`.

## 6. Content Model Direction

The content model is still block-based, but it now explicitly leaves room for richer study apparatus:

- canonical references per block
- tokenized text for word-level lookup
- phrase anchors for notes/highlights
- optional footnotes and textual variants
- imported dictionary entries and study indices

## 7. UI/UX Design System
- **Aesthetics:** Minimalist, clean typography, thoughtful whitespace, dark mode. Inspired by Claude Desktop, Arc Browser, native macOS.
- **Fonts:** Inter (UI), Merriweather/Newsreader/Crimson Text (reading).
- **Framework:** Tailwind CSS with design tokens in `src/index.css`.
- **Components:** Radix UI for accessible overlays, context menus, split panes.
- **Full spec:** See `docs/product/design_system.md`.
